# Learning Companion Project (LCP)

> A teachable learning companion for when learning isn't easy.

## Accessibility

A11y is very personal to me. I'm dyslexic — reading code is not easy
for me, let alone browsing websites. A lot of sites are overwhelming,
hard to navigate, hard to read, or just too loud. So for me, building
in A11y from the get-go is a necessity, not an exception.

### Who LCP is for

Right out of the box, LCP is for me — the person building it. But I
hope to expand it: for use by my children as a study buddy or tutor,
and eventually branching out to other areas — an Art companion, a
Writing companion, wherever the scope wants to grow.

### What's baked in

- **Terminal-first (for now)** — while I'm testing and iterating, a
  terminal interface keeps things simple and screen-reader-native. A
  UI is planned for later.
- **Line wrap at 72 characters** — dyslexia-informed; text wraps on
  word boundaries, not mid-word, and continuation lines align under
  the speaker prefix.
- **Graceful error handling** — no raw Python tracebacks dumped on
  you when something breaks. Errors surface as plain, actionable
  messages.
- **Persistent memory** — the conversation history persists across
  sessions, so you never have to re-explain yourself.

I'm proud of the rest too — no emoji by default, plain language over
jargon, task-focused not feelings-focused, no dark patterns, and
file-reference prompting so LCP never guesses what a file contains.

### My commitment

**A11y is baked in from day one, not bolted on later.**
