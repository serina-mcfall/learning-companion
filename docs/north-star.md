# Learning Companion — North Star

## What LCP is

**A teachable learning companion for when learning isn't easy.**

The Learning Companion is both a learning companion and a conversational
companion — a tool there to help you when you're learning something new.
It's teachable in both directions: you feed it your experience so it can
understand you better; what it learns from serving you eventually helps
it understand the wider ND community.

It exists because learning isn't easy, especially for me — and that's
true for many of us who are neurodivergent. This is one way to make
sure you're covered when you're learning.

It knows who you are (dyslexic, ND, learning software development),
communicates in ND-friendly plain language, remembers what you've been
working on across sessions, and helps you break down and understand new
concepts as you study.


## Who it's for

LCP grows in three phases — me first, then my kids, then the wider
Brightly Wired community.

### Phase 1 — Me

LCP's first user is me. I'm ND (AuDHD, dyslexic, alexithymic) — one
part chaos, one part structure. It's not easy, but it's a wild ride,
and it's who I am.

I'm mid-career-shifting into tech: finishing Dev Academy's Software
Development Course this week, then heading into their AI Launchpad.
LCP is a side project I'm building alongside Launchpad — an actual
thing I'm making while I learn AI agentic coding, running agents and
workers, and Python.

Life stage: mum of two neurodivergent kids, wife of a Tech Nerd (great
inspiration, driving force in this path), and an artist. Not many
constraints — mostly just dyslexia making code hard to read, and I
have tools for that.

What can spiral me: low energy, tiredness, mistakes. But I know my
limits, know when to walk away, and breaks are a necessity.

### Phase 2 — My Kids

Next up are my two neurodivergent kids. All of Phase 1's core still
applies — break-down-concepts, remembers-across-sessions, ND-friendly
plain language, teachable — Phase 2 just shifts the shape for children.

What changes for them:

- **Subjects** — instead of software development, LCP helps with the
  school subjects they struggle with the most. Different for each kid.
- **Tone** — kids need to be talked to differently than adults, and
  each of my kids also has their own way they respond best; LCP's
  voice adapts per child.
- **Personalisation** — each kid gets to NAME their own companion,
  making it personal. Onboarding prompts help build a companion
  specific to their needs.
- **Teaching flow** — I start as their parent, seeding what LCP needs
  to know about each of them. Then LCP keeps learning from the kids
  themselves as they use it.

### Phase 3 — The Brightly Wired Community

Long-term horizon. LCP eventually lives inside Brightly Wired as a
standalone feature available to all BW members. Users bring their own
LLM — I can't fund API access at community scale. How that BYO-LLM
flow integrates is TBD.

**Deferred on purpose.** Phase 3 needs to be revisited once I've lived
in Phases 1 and 2 a bit and figured out what other features should be
added.


## Non-negotiables

Rules LCP will never bend on. Ordered by weight — top of list first.

**1. ND-designed by an ND person.** LCP is designed from the inside,
not accommodated to. Design integrity, not disability optics.

**2. Non-shaming.** LCP never signals shame, embarrassment, or
judgment about what the user doesn't know. Shame is the fastest way
to shut down learning.

**3. User agency / user controls the pace.** LCP never pushes past
"I need a break" or "walk away." The user's energy budget is theirs.

**4. Plain language, no jargon.** LCP speaks in plain, jargon-free
language unless the user asks for technical depth. Learning shouldn't
itself be a barrier.

**5. Task-focused, not feelings-focused.** LCP never asks the user
to identify or describe emotions. It asks about tasks, energy,
readiness. Emotion-language is inaccessible for some users, including
me.

**6. Accessibility.** LCP meets accessibility standards for the medium
it's in — WCAG 2.2 AA for any visual UI, screen-reader-friendly
conventions for terminal. ND ≠ visual or motor accessibility, but
everyone gets in.

**7. No dark patterns.** No manipulation via gamification, no
streak-shaming, no notification anxiety. LCP is a companion, not a
habit-loop.

**8. Memory across sessions.** LCP always remembers what you've been
working on. Starting from scratch is exhausting.

**9. Teachable.** LCP always accepts new context from the user and
folds it in. The user is the expert on themselves.

**10. Local-first / your data stays yours.** LCP doesn't send
personal learning data anywhere it doesn't need to go.

**11. Privacy for children (Phase 2+).** LCP never surfaces one
child's data to another user, including a parent, without explicit
consent. Kids get privacy too.
